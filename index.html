The Core Scientific Claim

Core Proposition · Serrano (2026)

"The proportion of Theoretical issues in a scientific model's user community is a quantifiable, monotonic proxy for the scientific complexity frontier of that model — the boundary between what users can resolve through documentation and what requires original scientific investigation. This proportion decreases as model documentation matures, and increases as model domain complexity grows. Identifying the model-specific theoretical frontier defines the natural boundary of a research contribution."

Hypothesis Tests — All Four Models

H1 — Procedural Majority
Refined

Confirmed for SWAT family (48–55%) but NOT MODFLOW (28%). Procedural dominance is characteristic of management/interface-oriented models, not pure physics solvers. The distinction is domain type, not model age.

4-model evidence: domain type predicts procedural/theoretical ratio more than age.
H2 — Bug Rate vs Platform Youth
Confirmed

SWAT+ highest (12%) = newest + most interface integration points. SWAT-MF lowest (7%) = single-interface + focused developer support. Pattern holds across all 4 models.

Platform youth + interface complexity → higher bug rate. Confirmed.
H3 — Calibration Depth by Domain
Confirmed

MODFLOW calibration (PEST, pilot points) is qualitatively deeper than SWAT-family calibration (SWAT-CUP file formats, copy-back). Nature of T-09 issues differs: MODFLOW = theoretical; SWAT = procedural + theoretical.

4-model T-09 analysis confirms domain-dependent calibration depth.
H4 — T-CAT κ ≥ 0.75
Pending Formal Test

Automated classification rate >90% across ~9,500 threads. Formal inter-coder reliability (Cohen's κ) with a second independent coder is pending for the manuscript.

Preliminary automated tests across 4 communities show >90% classification rate.
H5 — Theoretical% Complexity Proxy
Confirmed (4 models)

MODFLOW (43%) > SWAT-MF (23%) > SWAT+ (21%) > SWAT2012 (18%) — perfect monotonic gradient confirmed with addition of SWAT2012. No counter-example found across all four models and ~9,500 threads.

Four-model gradient confirmed. Proposed as a general empirical law.
H6 — Model-Specific Frontiers
Confirmed (4 models)

Four distinct frontiers: SWAT2012 (sediment/nutrients), SWAT+ (decision tables), SWAT-MF (SW-GW exchange), MODFLOW (BC conceptualisation). Non-overlapping, model-specific, and each scientifically distinctive.

Each frontier maps to a specific T-CAT at HIGH frequency × HIGH theoretical rate.

Six Original Contributions

01
Universal T-CAT Taxonomy

12-category, 6-domain taxonomy validated across 4 models and ~9,500 threads. First cross-model taxonomy for hydrological model user issues.

02
Four-Way Nature Classification

Procedural / Theoretical / Config / Bug — with explicit, auditable decision rules. Enables quantitative comparison of any model community.

03
Theoretical% Complexity Law

Monotonic gradient confirmed across 4 models. Theoretical issue proportion is a quantifiable proxy for scientific complexity domain. Proposed as a general empirical law.

04
Complexity Frontier Concept

Each model has a unique frontier — the boundary between documentable knowledge and research gaps. Locating this frontier defines a PhD contribution's natural scope.

05
Diagnostic Algorithm

6-block systematic problem triage for SWAT-MF and SWAT+, derived from 9,500+ thread patterns. Reduces community support burden.

06
Reproducible Python Pipeline

Open-source coding pipeline (taxonomy.py + 02_code.py) enabling longitudinal monitoring and application to any new scientific model community.
