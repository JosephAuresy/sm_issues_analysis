Procedural vs. Theoretical — Key Distinction

Procedural issues arise from incorrect model setup, wrong file structure, mismatched simulation periods, or missing MODFLOW packages — they have definitive correct solutions documented in the manual.

Theoretical issues reflect gaps in understanding the underlying hydrology: how recharge is computed, what controls SW-GW exchange fluxes, or why a water balance closes differently in SWAT vs MODFLOW. These are scientifically substantive and merit dedicated research attention — they are the core of publishable contributions.

Issue Catalog — 8 Primary Categories

🔗
CAT-01 · Most Frequent
Model Linking & Coupling Setup
Procedural Config
120
threads

Problems with creating linkage files (hru_dhu, riv_grid, mf_riv, swatmf_link.txt), spatial intersection of SWAT HRUs with MODFLOW grid, and the QSWATMOD2 linking workflow. The most reported category by a wide margin.

Recurring examples
  • Missing HRU_ID field in shapefile; only HRUGIS present
  • river_grid shapefile fails to generate; Python errors in QSWATMOD
  • Intersecting DHRUs by GRIDs → processing not done
  • MODFLOW grid column/row count off by one
  • createSWATMF.exe usage unclear with non-QSWAT outputs
Systematic Solution

Standardize the HRU shapefile field validation step in QSWATMOD2. Implement pre-linking checker that validates all required fields, projection consistency, and grid extent overlap before proceeding. Provide a corrective auto-fix for HRU_ID generation.

⚠️
CAT-02 · High Priority
MODFLOW Convergence Failures
Theoretical Config
95
threads

MODFLOW fails to solve the groundwater flow equation within tolerance, causing the coupled simulation to crash or produce unreliable heads. Often linked to initial conditions, solver settings, dry cells, or inappropriate aquifer parameters.

Recurring examples
  • "FAILED TO MEET SOLVER CONVERGENCE CRITERIA" in *.lst
  • Water budget discrepancy > 1% indicating mass balance failure
  • Dry cell cycling leading to oscillation in NWT solver
  • Large hydraulic conductivity contrasts between adjacent cells
  • Initial head set to ground surface causing instant drainage
Systematic Solution

Develop a MODFLOW pre-run diagnostic: check water budget discrepancy, flag dry cells, validate HK transitions. Recommend MODFLOW-NWT over PCG2 for water-table scenarios. Provide parameter ranges (HK, SS, SY) typical for coupled SWAT-MODFLOW models.

💥
CAT-03 · High Frequency
Simulation Crashes & Runtime Errors
Procedural Bug
82
threads

The coupled executable crashes during or before simulation. Causes include mismatched SWAT/MODFLOW simulation periods, file indexing errors in modflow.mfn, missing output files, and array size limitations for large models.

Recurring examples
  • Crash at Day 1 or Year 1 / Period 1 — MODFLOW period shorter than SWAT
  • forrtl severe (29): file not found, fort.146
  • forrtl severe (43): file name specification error in modflow.mfn
  • Exit code 24 — end-of-file on .wel at Period 2
  • Access violation for large basins (>45,000 km²; 7500 HRUs)
Systematic Solution

Require MODFLOW simulation period ≥ SWAT period + 100 days. Enforce modflow.mfn indices >5000. Build a pre-run validator that checks period alignment and file index consistency. Document array size limits and compile notes for large-scale applications.

CAT-04 · Theoretical Core
Recharge & SW-GW Exchange Outputs
Theoretical Interpretation
68
threads

Misunderstanding or unexpected values in SWAT-MODFLOW's central exchange variables: recharge passed from SWAT HRUs to MODFLOW, and groundwater-surface water flux returned to SWAT channels. Core scientific content for PhD research.

Recurring examples
  • swatmf_out_MF_recharge_yearly shows implausible values for alluvial fans
  • SW-GW exchange limited to 2 cells instead of entire river reach
  • Missing swatmf_out_SWAT_gwsw file / incorrect output control flags
  • Huge recharge estimates post-run (parameter scale issue)
  • ET computed differently in output.std vs. expected values
Systematic Solution

Document the complete recharge transfer algorithm (HRU percolation → MODFLOW RCH package). Clarify that exchange cells are defined by river_grid intersection — limited exchange indicates spatial mismatch. Publish expected value ranges by climate/geology class. This category is a strong PhD paper candidate.

CAT-05 · Research Opportunity
Calibration & Parameter Sensitivity
Theoretical Procedural
53
threads

How to calibrate the coupled model (both streamflow and hydraulic head simultaneously), what parameters to prioritize, and how to use tools like SWAT-CUP or PEST with SWAT-MODFLOW. Joint calibration remains a key methodological challenge.

Recurring examples
  • How to link SWAT-MODFLOW with SWAT-CUP for calibration/validation
  • CN2 and Sol_Z (SWAT) vs. HK, SS, SY (MODFLOW) parameter interaction
  • High groundwater levels — which model side to adjust first
  • Multi-objective calibration: streamflow + GW head + ET + crop yield
  • GWCal tool applicability and workflow
Systematic Solution

Develop a standardized joint calibration protocol: (1) calibrate SWAT alone for streamflow, (2) extract recharge, (3) calibrate MODFLOW for GW heads, (4) run coupled model and iteratively refine. Integrate with QSWATMOD2 as a guided calibration workflow. Strong paper topic.

CAT-06 · Software Issues
QSWATMOD2 GUI & Plugin Errors
Bug Procedural
41
threads

Errors within the QSWATMOD2 QGIS plugin itself: Python exceptions, layer conflicts, version incompatibilities with QGIS, raster loading failures, and dhru.gpkg creation conflicts.

Recurring examples
  • dhru.gpkg already exists — file system conflict during linking
  • RT3D: cannot generate ICBOUND, multi-layer models not supported
  • mf_riv2 layer already on canvas when creating river_grid
  • Raster file loading fails — platform-specific (fixed in v2.10.0)
  • KeyError: 2 after simulation — output parsing error
Systematic Solution

Implement defensive file existence checks before all gpkg/shapefile creation steps. Add a "clean workspace" function. Maintain a tested compatibility matrix (QGIS version × QSWATMOD2 version). Track bugs in a public GitHub issue tracker to reduce Google Group noise.

⏱️
CAT-07 · Configuration
Temporal & Spatial Discretization Mismatch
Config Theoretical
33
threads

Mismatches between SWAT and MODFLOW discretization: MODFLOW stress periods not matching SWAT time steps, MODFLOW domain smaller than SWAT watershed, monthly/daily output scale issues.

Recurring examples
  • Stress period hang — NPER not updated in DIS file
  • MODFLOW period set to 517 days, SWAT to 3 years → crash at day 517
  • Monthly recharge output always fails — output control flag confusion
  • Smaller MODFLOW domain → no GW-SW interaction at boundaries
  • Long file path causes output file not found errors
Systematic Solution

Add a pre-run period alignment check: MODFLOW simulation end ≥ SWAT simulation end. Warn users when MODFLOW domain does not cover the full SWAT watershed. Document the swatmf_link.txt output control flags clearly in GUI with dropdown options.

CAT-08 · Advanced Use
Advanced Features (RT3D, Irrigation, Extensions)
Theoretical Procedural
17
threads

Questions about less-documented features: coupling MODFLOW pumping wells with SWAT irrigation routines, RT3D reactive transport integration, sub-hourly simulations, and SWAT-MODFLOW-SALT extensions.

Recurring examples
  • Linking MODFLOW WEL package to SWAT irrigation scheduling
  • RT3D ICBOUND generation fails; multi-layer RT3D not in QSWATMOD2
  • Sub-hourly pcp1.pcp format errors in SWAT-MODFLOW-SALT
  • Irrigation scenario configuration with coupled outputs
Systematic Solution

Create dedicated documentation for SWAT irrigation ↔ MODFLOW WEL coupling. Explicitly state RT3D multi-layer limitation in QSWATMOD2 docs. Develop a worked example for conjunctive use scenarios (pumping + irrigation + SW-GW feedback).
